# Decisions log

## 2026-07-03 — Sprint 0 skeleton
- Monorepo split into `api/`, `worker/`, `frontend/`, `core/` (empty, shared package for Sprint 1+), `infra/` (empty, Sprint 5).
- FastAPI settings via `pydantic-settings`, env-driven, no hardcoded config.
- Worker runs APScheduler `BackgroundScheduler` in a simple sleep loop rather than a full async event loop; matches the "heartbeat + scheduled jobs" shape needed through Sprint 3 without pulling in extra infra.
- `mypy --strict` on both Python services; added a `tool.mypy.overrides` ignore for `apscheduler.*` since it ships no type stubs, rather than weakening strictness project-wide.
- Frontend scaffolded with `create-next-app` (App Router, TypeScript strict, Tailwind, ESLint) and immediately given a real visual identity (dark terminal aesthetic, IBM Plex Sans/Mono, OKLCH tinted neutrals) instead of shipping the default template — a deliberate scope pull-forward from Sprint 4, approved by the user mid-session. Full page content (dashboard, chat, briefs) still waits for Sprint 4.
- Compose healthchecks: `pg_isready` for postgres, `redis-cli ping` for redis, a bare TCP probe for qdrant (no curl/wget in its image), a Python urllib probe for the API, and a Docker-level `HEALTHCHECK` in the worker's Dockerfile (no HTTP surface to probe otherwise).

## 2026-07-03 — Sprint 1 market and on-chain data layer
- `core/` is now a real installable package (`argus-core`) holding config, db, models, data clients, cache, repository and the Alembic tree. Both `api` and `worker` depend on it. To make it importable inside each image, the compose build context moved from `./api` / `./worker` to the repo root (`context: .`, `dockerfile: <svc>/Dockerfile`); each Dockerfile installs `./core` first, then the service. This is the standard monorepo shared-package pattern, not a deviation from CLAUDE.md's mandated `core/`.
- Added `argus_core/py.typed` so `mypy --strict` in api/worker treats the shared package as typed rather than erroring on a missing marker.
- Async everywhere: SQLAlchemy 2.0 + asyncpg, async FastAPI sessions. Worker jobs are sync (BackgroundScheduler) and wrap the async collectors in `asyncio.run`, avoiding a second sync DB stack. DATABASE_URL therefore uses the `postgresql+asyncpg://` driver.
- HTTP clients share a `BaseClient`: httpx async, tenacity exponential backoff (retries transport errors and non-2xx incl. 429), and opt-in Redis caching with hard TTLs (CoinGecko current price 60s, historical charts 1h; DeFiLlama 1h; mempool 60s) to stay under CoinGecko's 30 calls/min free limit.
- Snapshot idempotency via Postgres `INSERT ... ON CONFLICT DO NOTHING` on `(coingecko_id, ts)` and `(defillama_slug, ts)` unique constraints, so re-runs and the backfill never duplicate rows.
- Exit test needs 30 days of history immediately, but snapshot jobs only accumulate going forward. Added an idempotent `worker/app/backfill.py` (`python -m app.backfill`) that pulls 30 days of daily price/TVL history from the free historical endpoints, seeding the tables up front.
- Alembic env runs async (`async_engine_from_config` + `run_sync`); the worker applies `alembic upgrade head` on container start before scheduling, so migrations own the schema from the first table.
- Auth is a single-user bearer token dependency applied per-router (`/health` stays open). Designed to swap for real auth later without touching handlers.
- CI gained a `core` job; api/worker jobs `pip install ../core` before their own install. Also added `master` to the push trigger (repo default branch is `master`, Sprint 0 only listed `main`).
- Rate-limit hardening (found during exit-test verification): CoinGecko's free tier 429'd the backfill because it fired 8 chart calls in a burst; its real burst tolerance is well under the nominal 30/min. Fixed by (a) longer retry backoff in `BaseClient` (6 attempts, exponential to 60s) and (b) pacing the backfill 8s between CoinGecko calls. Exit test then returned 31 daily ETH price points, 31 daily AAVE TVL points, and live BTC mempool fees.

## 2026-07-03 — Sprint 2 regulatory RAG pipeline
- **fastembed instead of sentence-transformers** for bge-small-en-v1.5: identical model and vectors, ONNX runtime instead of PyTorch, keeping images ~100MB instead of ~2GB — matters on a free ARM VM. Disclosed deviation from CLAUDE.md's "sentence-transformers" wording; the requirement it encodes (local, zero-cost bge-small embeddings) is unchanged.
- Query-time embedding runs **in the API process** (fastembed is a core dep), not via a call to the worker: simplest topology, and the model is small enough to load in both containers.
- MiCA chunking: EUR-Lex official HTML wraps each article in `div.eli-subdivision#art_N` — exactly 149 of them, matching MiCA's known article total, so article divs are the lossless chunk boundary. Long articles split at paragraph boundaries (~2400 chars ≈ bge-small's 512-token window) with the article ref preserved on every part. Tests pin the 149 count and spot-check Articles 4, 6, 36 against the committed EUR-Lex fixture.
- FINMA PDFs (ICO guidelines 2018, stablecoin supplement 2019) verified to have real text layers (~18k/~13k chars via pypdf); split on numbered-section headings, jurisdiction CH. Ingestion logs a 300-char extraction sample each run so a silently-scanned replacement PDF would be caught by eye.
- Whitepapers: BTC/AAVE/UNI/LINK PDFs split on numbered headings, ETH (HTML) on h2/h3; sections under 80 chars dropped as TOC artifacts. One unreachable source skips with a warning rather than sinking the run.
- Qdrant `regulatory` collection: 384-dim cosine, payload indexes on source/jurisdiction/ref, deterministic point IDs (sha256 of source|document|ref|part) so re-ingestion is an idempotent upsert. Weekly re-ingest job; content-hash diffing deferred.
- LLM behind `core/llm.py`: provider chain from `LLM_PROVIDER` + configured keys (Gemini free tier primary, Groq fallback, Anthropic opt-in). Prompts are versioned files in `argus_core/prompts/` shipped as package data.
- `/research/ask`: LLM query rewrite (best-effort; falls back to the raw question), top-12 retrieval with optional jurisdiction filter, **score cutoff 0.62**, calibrated against the live corpus (on-topic regulatory questions score ≥ 0.73 with bge-small cosine, off-topic questions ≤ 0.58; the initial guess of 0.45 let "best pizza in Zug" through) — below it the endpoint refuses honestly with no LLM call. Answers require inline [n] citations; citations echoed in the response are restricted to the [n] actually present in the answer. Every query/retrieved-set/answer logged to `rag_queries` with latency.
- Hard no-advice rule lives in the system prompt (`rag_answer_v1.txt`): explain what regulations say, never advise.

## 2026-07-11 — Sprint 3 agent layer
- **`daily_brief`**: assembles structured data from Postgres (24h watchlist movers, 24h TVL shifts, unlocks in the next 7 days) plus one retrieval pass over the `regulatory` collection, then hands the whole data block to `core/llm.py` with a versioned template (`daily_brief_v1.txt`). The prompt is constrained to use only the provided numbers and to explain-never-advise, mirroring the RAG rule — the brief is the other place a confident hallucinated number would do damage. Regulatory pass is best-effort (a Qdrant/embedding failure degrades the section, doesn't sink the brief).
- **One brief per calendar date**: `brief_date` is normalised to midnight UTC so the unique constraint actually dedupes per day; re-runs on the same day upsert (`ON CONFLICT DO UPDATE`) rather than piling up rows. Scheduled 07:00 Europe/Zurich (handles CET/CEST).
- **Email over SMTP** (Gmail app password) via `core/email.py`, plain-text Markdown body. Missing credentials degrade to a logged skip, not a crash, so the brief is still stored. Resend is a drop-in swap through the same `smtp_*` settings.
- **`anomaly_scan` z-scores computed in Python, not pure SQL** — disclosed deviation from CLAUDE.md's "z-scores in SQL". The 30-day series is pulled with portable SQL and mean/std/z computed in Python (`statistics.pstdev`). Reason: the test suite runs on in-memory SQLite, which has no `stddev`/window-stats functions, so a Postgres-native window query couldn't be unit-tested without adding a testcontainers-Postgres dependency. The requirement the wording encodes (rolling 30-day z-scores on returns/volume/TVL deltas, |z|>3) is unchanged and fully covered by tests.
- Anomaly **dedupe** is a portable "fetch existing keys, insert only new" step keyed on `kind:symbol:metric:day` (not a Postgres `ON CONFLICT`), so the same spike doesn't refire every 30 minutes and the logic is SQLite-testable. Depeg rule: stablecoins only, `|price-1|>0.005` sustained across the last 3 snapshots.
- **Incremental ingestion**: each Qdrant point now carries a `content_hash` of its text; `existing_content_hashes()` scrolls them back and `embed_and_upsert` re-embeds only chunks whose text changed. This is the content-hash diffing that Sprint 2 deferred — the weekly job no longer re-embeds the whole corpus.
- **Manual ingest trigger without cross-service coupling**: `POST /admin/ingest` (API) sets a Redis flag; a 1-minute worker job (`check_ingest_request`) atomically pops it (`GETDEL`) and runs ingestion. The embedding model stays out of the API image — the clean api/worker split is preserved at the cost of ≤60s latency, an acceptable trade for a manual admin action.

## 2026-07-18 — Sprint 4 frontend
- **Design direction "Regulatory Terminal — a Swiss ledger in the dark"**: Ink/Slate/Parchment/Seal/Glacier palette, Spectral serif (prose/headings) + IBM Plex Mono (all data), on a faint ledger-grid background. Signature element is the **citation seal + jurisdiction rail** — citations render as numbered chips carrying document, article/section ref and an EU/CH provenance glyph linking to the real EUR-Lex/FINMA URL, and a persistent left rail marks EU-blue → CH-amber provenance. Chosen over generic dark-crypto-dashboard defaults because the target audience (SDX/Sygnum/FINMA) reads legal registers, and Swiss-grid precision fits literally.
- **Stack is Next 16 / React 19 / Tailwind v4** (App Router), not the "Next.js 14" CLAUDE.md originally said — the scaffold was already 16 and there was no reason to downgrade; corrected the CLAUDE.md line.
- **Token stays server-side**: `src/lib/api.ts` reads `ARGUS_API_URL`/`ARGUS_API_TOKEN` (no `NEXT_PUBLIC_`). Server Components fetch directly; the client research chat posts through a Route Handler proxy (`app/api/research/route.ts`) so the bearer token never reaches the browser.
- **Added `GET /market/unlocks`** (small backend addition): the `unlock_events` table + `UnlockEvent` model existed and the daily brief read them, but no endpoint exposed upcoming unlocks. Added it (joined to watchlist symbol/name, next-N-days) so the dashboard unlock calendar shows real curated data instead of a stub. Disclosed deviation from "frontend-only" sprint scope; completing the dashboard spec honestly was the better call.
- **Chat reveal is a client-side typewriter over the complete returned answer, not token streaming** — `/research/ask` returns a full `ResearchAnswer` in one response; labelled honestly in UI copy and code comments, collapses to instant under `prefers-reduced-motion`. True SSE streaming deferred (backend change, out of scope).
- **Sparklines hand-rolled as SVG** (no charting dependency), colour follows trend. Same minimal-deps posture drove a small dependency-free markdown renderer for brief bodies and plain controlled form state (no react-hook-form/zod) for the single-field research box.
- **Anomaly filters via URL search params** (server-rendered, shareable) rather than client fetching. Every page carries the disclaimer footer; visible `:focus-visible` rings and reduced-motion handling are global in `globals.css`.
- Every page degrades honestly when the API is down (unreachable banner / empty states) — the watchlist fetch doubles as the reachability probe on the dashboard. Verified all routes return 200 with the backend offline; proxy returns 502 (backend down) and 400 (empty question).

## 2026-07-18 — Sprint 5 deployment + IaC (host-agnostic phase)
Oracle signup was blocked pending a support inquiry, so this sprint delivered everything that does **not** need a live VM IP; nothing in the repo names a host.

- **Host-agnostic by construction**: `deploy.yml`/`infra/deploy.sh` take host, user, key and domain from secrets/env; `docker-compose.prod.yml` swaps `build:` for GHCR `image:` refs (`GHCR_OWNER`/`IMAGE_TAG` vars); the Caddyfile reads `ARGUS_DOMAIN`/`ARGUS_ACME_EMAIL` from the environment. Switching Oracle → Hetzner → EC2 is a secrets change, not a code change — this is also the Oracle-capacity fallback insurance.
- **Multi-arch images** (linux/amd64 + linux/arm64) via buildx matrix in `release.yml`, GHA layer caching per image, tags `latest` + short SHA. arm64 exists solely for the Oracle Always Free ARM VM.
- **Health gate is the deploy contract**: the workflow polls `https://$DOMAIN/health` (raw `host:8000` fallback pre-DNS) for up to 2 minutes and fails the run on non-200 — a deploy that doesn't serve traffic is a failed deploy, visibly.
- **Frontend production image** (Next standalone, non-root, ~small) added even though Vercel stays the primary frontend host: it makes the stack fully self-hostable behind Caddy (Cloudflare-Tunnel fallback needs it).
- **Backups with a tested restore**: nightly `pg_dump -Fc` + per-collection Qdrant snapshot API, gzip + datestamp, local retention pruning, offsite via a pluggable `BACKUP_TARGET` — private GitHub release (default, zero-cost) or any S3-compatible endpoint (covers Oracle Object Storage later). `restore.sh` ships alongside because an untested backup isn't one.
- **CloudFormation stays a portfolio artifact**: single-instance template (VPC, public subnet, SG 22/80/443, arm64 Ubuntu via SSM public AMI parameter, Docker UserData). cfn-lint + shellcheck run in a new CI `infra` job, since no AWS credentials exist locally; the one live end-to-end AWS run (with screenshots and timing) is deferred to the AWS-trial session.
- **Uptime Kuma** in the prod overlay at `status.$ARGUS_DOMAIN`; monitors are configured in its UI post-deploy.
- `.gitattributes` forces LF + executable bit on all shell scripts so a Windows checkout can't ship CRLF scripts to a Linux host.

### Remaining manual steps once a host is available
1. Create the VM (Oracle A1.Flex when unblocked, or any Ubuntu host). Open 22/80/443 in the provider firewall (Oracle: VCN security list, ufw alone is not enough).
2. Copy `infra/host-setup.sh` to the VM and run it; log out/in for the docker group.
3. Copy `infra/.env.prod.example` to `~/argus/.env` on the VM and fill in real values.
4. `docker login ghcr.io` on the VM with a read-only PAT (or make the GHCR packages public).
5. Point DNS (DuckDNS subdomain) at the VM's public IP.
6. Add GitHub secrets: `DEPLOY_HOST`, `DEPLOY_USER`, `DEPLOY_SSH_KEY`, `DEPLOY_DOMAIN`.
7. Run the **Deploy** workflow once by hand (workflow_dispatch) and confirm the health gate passes green.
8. Add the backup cron line (in `infra/backup/backup.sh` header) and run one backup + one restore drill.
9. Configure Uptime Kuma monitors at `status.$DOMAIN` (API `/health`, frontend, Vercel URL).

## 2026-07-21 — Sprint 6 polish + portfolio packaging
- **Citation eval reports two numbers, not one** (`eval/run_eval.py`): a lenient **hit rate** (expected source cited anywhere) and a strict **top-1 accuracy** (expected source cited first). Publishing both is more honest and more interesting to defend than either alone — it separates "retrieval found the right document" from "the single best-ranked citation was exactly right". First live run over 18 questions (10 MiCA article-level, 5 FINMA, 3 whitepaper): **83.3% lenient (15/18), 55.6% top-1 (10/18), 0 refusals**. The number goes in the README as-is.
- **Eval matching is document + optional ref-substring**, normalised so "Article 6" == "Art. 6". MiCA cases pin an exact article; FINMA/whitepaper cases assert document-level (their section numbering / heading text is brittle to pin, and the honest expectation there is "right document, right jurisdiction"). Lenient misses (rephrased Art. 6, ART Art. 16, EMT Art. 48) cited **adjacent** MiCA articles rather than refusing — a real, reportable retrieval-ranking limitation, not hidden.
- **Eval robustness over the free-tier LLM**: the harness paces requests (5s) and retries transient 503s with backoff, because the free Gemini/Groq per-minute limit otherwise throttles ~half the run and would masquerade as citation misses. It exits non-zero if the backend is unreachable, so a broken run can never be mistaken for a 0% score. `sys.stdout.reconfigure(utf-8)` because the Windows dev console is cp1252 and choked on the ✓ glyphs.
- **Grid texture reduced** (`globals.css`): the ledger grid read as a templated dark-mode default. Lines diluted to ~32% toward the background, cell enlarged 56px → 88px, vignette deepened. The Swiss-ledger signature stays but registers as faint paper texture, not a visible table.
- **README rewritten as the portfolio front door** (Sprint 6 exit test = a cold technical reader runs it from the README alone): one-line pitch + live-URL placeholder, Mermaid architecture diagram (renders on GitHub), four-page screenshot grid (placeholders under `docs/screenshots/`), the measured citation numbers, design-decisions/tradeoffs distilled from this log, a "what I'd change at 100x scale" section, and clone→compose→ingest setup. The AI-assisted-development disclosure explicitly states citation accuracy is **measured** via the eval harness against the live backend, not asserted — the sentence that separates a credible disclosure from a generic one.
- **Portfolio + LinkedIn copy kept as repo artifacts** (`docs/portfolio-copy.md`, `docs/linkedin-post.md`) so the positioning is versioned alongside the code it describes rather than living only in a profile doc.
